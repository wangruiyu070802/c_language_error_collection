## **第四部分：图片如何正确显示**

GitHub 的 Markdown Preview 显示图片有两个关键点：

**方式一：存在仓库 `assets/` 文件夹里（推荐）**

图片放在 `assets/` 文件夹，用**相对路径**引用：

```markdown
![图片描述](../../assets/ptr_01.png)
```

路径里的 `../` 数量取决于当前文件的层级深度，从 `01_指针/q01_野指针/question.md` 回到根目录需要两个 `../`。

**方式二：直接上传到 GitHub 后用绝对链接**

在 GitHub 网页上打开任意一个 `.md` 文件，点编辑，然后**直接拖拽图片**进去，GitHub 会自动上传并生成一个 `https://github.com/user-attachments/...` 的链接，这种方式最省事，不用管路径。

## **第五部分：实际操作流程**

每次新增一道题，只需要三步：

```bash
# 1. 把图片放进 assets/ 文件夹

# 2. 在对应分类下新建文件夹和 md 文件
# 例如新增一道数组越界题：

# 然后写好 question.md 和 answer.md

# 3. 提交推送

git commit -m "新增：03_数组/q01_数组越界"
git push
```
